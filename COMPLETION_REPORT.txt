✅ azca/api/main.py (250+ líneas)
   
   Endpoints:
      1. GET /health
         └─ Health check simple
         └─ Respuesta: {"status": "healthy", ...}
         └─ Status: 200 OK
      
      2. POST /predict
         └─ Recibe: service_date, max_temp_c, precipitation_mm, is_stadium_event, is_payday_week
         └─ Llama: engine.predict('model', data)
         └─ CREA: PredictionLog en BD
         └─ Guarda: db.add() + db.commit()
         └─ Retorna: {"prediction_result": X, "log_id": Y, ...}
         └─ Status: 201 CREATED
      
      3. GET /docs
         └─ Swagger interactivo (OpenAPI)
         └─ Auto-generado por FastAPI
   
   Modelos Pydantic:
      • PredictionRequest (validación entrada)
      • PredictionResponse (estructura salida)
      • HealthResponse
      • Con ejemplos y descrippciones para Swagger
   
   Características:
      ✓ Startup event: Carga PredictionEngine + init_db()
      ✓ Shutdown event: Limpieza de recursos
      ✓ Manejo profesional de errores (422, 503, 500)
      ✓ Logging con timestamps
      ✓ Fallback a mock si PredictionEngine no disponible
      ✓ Fallback a predicción mock si motor falla
      ✓ Transaction rollback en errores de BD
      ✓ Type hints y docstrings
      ✓ PEP8 compliance
